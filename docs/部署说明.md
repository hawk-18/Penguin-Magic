# 部署与运行说明

## 启动方式

在项目根目录（与 `Start.bat`、`Restart.bat` 同级）下：

- **首次安装或需要重装依赖并重新构建前端**：双击运行 **`Restart.bat`**。脚本会停止已有服务、安装前后端依赖、执行前端构建，再启动 Node 后端并打开浏览器（默认 `http://127.0.0.1:8765`）。
- **日常启动**（已完成 `Install.bat` 且存在构建产物 `dist`）：双击运行 **`Start.bat`**。脚本会清理占用端口的旧进程、启动后端并打开浏览器。

> **若启动失败**：请 **再执行一次** 同一脚本（`Start` 或 `Restart`）。偶发端口占用、依赖未就绪或构建未完成时，第二次重试常会成功。

其他脚本（供参考）：

- `Install.bat`：安装依赖并构建，适合环境准备阶段。
- `Stop.bat`：停止后台服务。

运行前请确保本机已安装 **Node.js**（建议 18 及以上），并按脚本提示完成依赖与构建。

---

## 界面异常：顶部导航栏无法点击

若出现 **顶栏标签（如 Chat / 桌面 / 画布）点不到** 的情况，可尝试：

1. **稍等片刻**：界面或 Electron 层完全加载后再点击。
2. **将窗口缩小一些**：避免全屏或过大窗口下层级/命中区域异常（部分环境下窗口尺寸会影响可点击区域）。

---

## 对话与媒体生成失败

若 **对话、图像生成、视频生成** 请求失败或报错，除网络与配置问题外，还可能是：

- **API 用量或配额已达上限**（第三方接口、模型额度等）。

请检查后台配置、密钥与服务商控制台中的用量说明。

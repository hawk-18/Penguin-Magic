# Chat 与画布界面面板说明

本文档描述主应用中 **Chat** 视图与 **画布（PebblingCanvas）** 视图的主要界面区域、面板与交互逻辑，便于开发与使用参考。

对应源码目录：

- Chat：`components/Chat/ChatPage.tsx`
- 画布：`components/PebblingCanvas/`（入口 `index.tsx`）

---

## 一、Chat 界面

### 1.1 整体结构

Chat 为独立全页视图（应用顶栏可切换到「Chat」）。典型布局为：

- **左侧**：会话列表（历史对话、新建、置顶、重命名、删除等）。
- **中间**：当前会话的消息流（用户气泡 / 助手气泡）。
- **底部**：复合输入区（附件条 + 多行输入 + 工具条）。

数据通过后端 `services/api/ai` 中的会话接口（如 `getChatSessions`、`createChatSession`、`addChatMessage` 等）持久化；流式对话走 `/api/ai/chat`。

### 1.2 会话侧栏

- 展示全部会话，支持切换当前会话。
- 常见操作：新建会话、置顶/取消置顶、重命名、删除；部分条目带三点菜单。
- 无会话时会自动创建「新对话」并选中。

### 1.3 消息区

- **文本消息**：支持轻量 Markdown（如粗体、列表等）与代码块高亮展示；代码块可复制，可进入全屏阅读。
- **图像消息**：展示生成结果或引用图；生成中有占位与进度相关展示。
- **视频消息**：内嵌播放器；任务状态（排队、渲染中等）有中文状态文案；完成后可播放、复制链接、重新生成等。
- **跟进问题**：部分助手文本回复下方会展示 2 条「可能继续问」的短问题（由模型生成或规则兜底），点击可填入输入框。

### 1.4 输入区与附件

- 主输入为 **textarea**：`Enter` 发送，`Shift+Enter` 换行；未聚焦时空占位文案随当前「输入模式」变化。
- **附件**：支持图片、视频（体积有上限）、文档（如 txt/docx，文本会抽取并参与上下文，有字符上限）。附件以缩略图条展示在输入框上方，可单独移除。
- **参考链接**：可提供引用链接（具体 UI 以代码为准），用于扩展上下文。

### 1.5 输入模式（技能）

通过底部工具条或 **`/` 快捷菜单** 可切换模式（`InputMode`），包括：

| 模式 | 含义 |
|------|------|
| 普通对话 | 默认聊天，无模式标签 |
| 帮我写作 | 长文/文案类创作 |
| 编程 | 代码解答与编辑 |
| PPT 生成 | 幻灯片相关生成（可与配图能力组合） |
| 翻译 | 需选择目标语言（简体/繁体/英文等） |
| 数据分析 | 数据与图表解读 |
| 图像生成 | 文生图等，可选画面比例、参考图 |
| 视频生成 | 视频描述生成，可选比例、参考图 |

**`/` 快捷指令**：在输入中输入 `/` 会弹出「快捷功能」列表，可用关键字过滤；方向键与 Enter 选择，Esc 取消。选中后会清除 `/...` 片段并设置对应 `inputMode`。

未手动选模式时，系统也可根据用户正文 **推断** 图像/视频意图并直连对应生成接口（见 `inferMediaIntentFromUserText`）。

### 1.6 快速 / 深度思考

- **快速**：默认系统提示，回答偏简洁可执行。
- **深度思考**：系统提示鼓励分步推理、假设与分支后再给结论（`chat_think`）。

### 1.7 图像与视频比例

在「图像生成」或「视频生成」模式下，可通过 **比例** 按钮打开上拉菜单，选择如 `1:1`、`16:9`、`9:16` 等。图像侧会映射为具体像素尺寸（`mediaAspectToImageSize`），视频侧用于任务参数。

### 1.8 助手消息操作条

对助手消息常见操作包括：复制、重新生成、朗读、点赞/点踩（可展开原因标签并提交）、分享入口、更多菜单中的删除等。反馈面板为共用组件样式。

### 1.9 短时记忆

多轮对话会将近期用户/助手内容压缩进请求上下文（有轮数与字符上限），避免单轮过长；实现见 `SHORT_TERM_MEMORY_*` 与 `uiMessagesToMemoryPayload`。

## 二、画布界面：图像类能力说明

本节**仅**说明画布中与 **`ImageSuggestionPanel`** 对应的五项能力（侧栏 **Image** 悬停面板与画布 **图片节点** 输出侧「+」菜单为同一套入口）。实现见 `components/PebblingCanvas/ImageSuggestionPanel.tsx`、`Sidebar.tsx`、`index.tsx`。

### 2.1 如何打开与共用流程

- **入口一**：左侧栏 **Image** 按钮悬停出现的「尝试」列表。  
- **入口二**：画布上 **图片节点** 输出端「+」打开的同款列表。  
- 选中任一项后，会打开 **可拖拽的浮动生成面板**；必要时会在画布上生成 **composer-anchor** 锚点，用于把上游参考图 **连线** 到面板。

**参考图连线（重要）**：**图片换背景**、**图片扩展**、**首帧图生视频** 在逻辑上 **必须** 将画布上的图片节点连到浮动面板左侧锚点后再生成；未连线时应用会提示先完成连线。图生图、图生视频在已选参考图或已连线时按图生链路处理。

浮动面板内可配置画质、比例、张数、底模（如豆包 / ComfyUI）及视频相关参数等；生成结果在面板内预览，可与全局「生成中」状态联动。

### 2.2 图生图（`native-image`）

- **含义**：在 **参考图** 基础上，按输入的文本描述生成新图（图生图）。  
- **使用**：从上述入口选择「图生图」，在浮动面板输入描述并提交；建议将源图连至锚点或从节点菜单带入参考图，以保证参考一致。

### 2.3 图生视频（`native-video`）

- **含义**：可带参考图，按描述生成 **动态视频**（与「首帧图生视频」链路不同，偏整体图生视频）。  
- **使用**：选择「图生视频」，在面板中填写画面/动作描述，并按需连接参考图、设置时长等参数后生成。

### 2.4 图片换背景（`remove-bg`）

- **含义**：**替换背景** 为描述中的场景，尽量 **保留清晰主体与边缘自然**。  
- **使用**：必须先将作为主体的 **图片节点连线到浮动面板锚点**，再输入希望的场景描述并生成。  
- **默认提示倾向**（可在面板中改写）：强调替换背景、保留主体与边缘自然。

### 2.5 首帧图生视频（`enhance-details`）

- **含义**：以 **首帧画面** 为基准，生成 **连贯、自然的动态与镜头运动**（首帧驱动的视频）。  
- **使用**：**必须** 连线提供首帧参考图后再生成；时长等选项与是否已连参考图相关（应用内会按接口能力限制可选秒数）。

### 2.6 图片扩展（`expand-image`）

- **含义**：**向外扩展画面边缘**，使构图更完整，并保持 **风格与透视一致**。  
- **使用**：**必须** 将待扩展的 **图片节点连线到锚点**，再补充描述（可选）并生成。  
- **默认提示倾向**（可在面板中改写）：自然扩展边缘、风格与透视一致。
